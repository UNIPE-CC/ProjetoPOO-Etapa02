## branch
#main
- Terá apenas o código FUNCIONAL E FINAL !!!

#dev
- Será a subMain
- Aqui conterá as features implementadas e funcionais.
- Recebe o merge da suaBranch

#branch pessoal
- Cada um cria a sua branch.
- A branch criada será reposnsável pelos seus commits pessoais.
- Os commits serão seus refatoramentos do código base (fork), features necessárias ao desenvolvimento.


#Guia do time
- 1 - clone o repositório a sua maquina física. IDE de sua preferência.
- git init
- git clone "url"

- 2 - Crie a sua branch

- 3 - Branch criada
- git status
- git pull

- 3.1 - Branch desatualizada
- git status
- git pull dev

- 4 - De commits na sua branch (criada)
- -- git add .
- -- git commit -m "comentário"
- -- git push suaBranch
- teste o programa
- - se não funcionou, não suba para a dev
- - Se estiver OK
- git status dev
- 
- 4.1 desatualizada - passo 3.1
- teste o programa
- - - se não funcionou, não suba para a dev
- - Se estiver OK
- git status dev
- 
- 4.2 atualizada
- teste o programa
- - OK. de um merge na branch dev

- 5 - Branch dev OK, subimos para a main
